# Auto-Lister

Upload a product image (or paste a barcode, or a URL) and get a listing ready for **Shopify** or **eBay** — SEO-optimised HTML description, a suggested AUD price, image alt text, and platform-specific fields — generated by AI.

Switch between the **Shopify** and **eBay** tabs to control which platform's output the AI produces and where it pushes.

Runs entirely on your machine with a free local model (Ollama), or connect a Claude API key for cloud-speed generation with image recognition support.

---

## Setup & running

### Step 1 — Install Node.js

Node.js is the runtime the app needs. You only do this once.

- Go to **[nodejs.org](https://nodejs.org)** and download the **LTS** version
- Run the installer and keep all default options
- Restart your PC/Mac after installing

To verify it worked, open a terminal and run `node --version` — you should see a version number.

---

### Step 2 — Choose an AI provider

The app needs an AI to generate the product descriptions. Pick one:

#### Option A — Ollama (free, runs on your computer)

Best if you want no ongoing cost and don't mind a 2–3 minute generation time on CPU (much faster with an NVIDIA GPU).

1. Go to **[ollama.com](https://ollama.com)** and download Ollama for your OS
2. Install it, then open a terminal and run:
   ```bash
   ollama pull qwen3:8b
   ```
   This downloads the AI model (~5 GB, one-time download).
3. Keep Ollama running with:
   ```bash
   ollama serve
   ```
   You'll need this running whenever you use the app.

> **Note:** Ollama cannot analyse product images — for image-only input, use Claude API (Option B).

#### Option B — Claude API (cloud, ~5 seconds per listing)

Best if you want fast results or need image recognition. Uses Anthropic's servers — costs a small amount per listing (typically fractions of a cent).

1. Go to **[console.anthropic.com](https://console.anthropic.com)** and create an account
2. Go to **API Keys** and create a new key (starts with `sk-ant-...`)
3. Enter the key in the app's **Settings → AI Provider** panel — no file editing needed

---

### Step 3 — (eBay only) Get eBay API credentials

Skip this step if you only need Shopify listings.

To push draft listings to eBay, you need credentials from the eBay Developer Programme:

1. Sign up at **[developer.ebay.com](https://developer.ebay.com)**
2. Create an app under **My Account → Application Keysets** — choose **Production**
3. Copy your **App ID (Client ID)**, **Dev ID**, and **Cert ID (Client Secret)**
4. Add them to your `.env` file (see Environment variables below)
5. Get your **User Token** (OAuth refresh token or legacy Auth'n'Auth token) from the eBay developer portal — enter it in the app's **Settings → eBay** panel

---

### Step 4 — Download and run the app

1. Download this repo — click the green **Code** button on GitHub → **Download ZIP**, then extract it  
   *(or if you have Git: `git clone https://github.com/Sweevee1/ShopifyAutoLister.git`)*

2. Open the extracted folder and launch:

   **Windows** — double-click **`Start.bat`** to start, **`Stop.bat`** to stop

   **Mac / Linux** — open a terminal in the folder and run:
   ```bash
   chmod +x start.sh
   ./start.sh
   ```

The launcher will install dependencies automatically on first run (takes about a minute), then open the app in your browser.

If you chose **Option B (Claude API)**, open **Settings**, expand **AI Provider**, select **Claude API**, and paste your key.

---

> **Every time you want to use the app:** just double-click `Start.bat` (Windows) or run `./start.sh` (Mac/Linux). If you're using Ollama, make sure `ollama serve` is running in a separate terminal.

## How to use it

Open [http://localhost:3000](http://localhost:3000).

**1. Choose your platform** — click the **Shopify** or **eBay** tab at the top.

**2. Provide product input** — give it one or more of:

| Input | Notes |
|---|---|
| **Product image** | Drop or browse — resized to 800 px before sending. Claude API can identify the product from the image alone. |
| **Barcode** (UPC / EAN / ISBN) | Looks up name and brand automatically via UPC Item DB |
| **Official product page URL** | Skips lookup/search; scrapes the URL directly |
| **Pasted page HTML** | Fallback when the live URL is blocked or JS-rendered |

You can combine any of these — e.g. an image **and** a barcode, or a barcode **and** a URL.

**3. Click Look Up Product.** The AI streams the result token-by-token.

**4. Push or copy:**
- **Shopify tab** — review the HTML description and price, then click **Push to Shopify** to create a draft product in your store. Review and publish from the Shopify admin.
- **eBay tab** — review the title (80-char limit), description, condition, item specifics, and category (auto-suggested from the product name), then click **Push to eBay** to save a scheduled listing in Seller Hub. The listing is scheduled 30 days out so it stays as a draft — activate it from Seller Hub when ready.

## AI provider

### Ollama (default — free, local)

Runs `qwen3:8b` on your own hardware. No API key, no cost, completely private. Slow on CPU-only machines (2–3 min), fast on a GPU.

```bash
ollama pull qwen3:8b
ollama serve
```

**PC requirements:**
- 8 GB RAM minimum (16 GB recommended)
- ~6 GB storage for model weights
- NVIDIA GPU with 8 GB+ VRAM strongly recommended for fast inference

### Claude API (optional — cloud, fast)

Enter your Anthropic API key in **Settings → AI Provider → Claude API**. Generation takes a few seconds instead of minutes. The key is stored in your browser only.

Get a key at [console.anthropic.com](https://console.anthropic.com). Image analysis (identifying a product from a photo) only works with Claude API — Ollama has no vision support.

## External services

| Service | Purpose | Cost | Key required |
|---|---|---|---|
| **UPC Item DB** | Barcode → product name, brand, price range | Free, 100 req/day | No |
| **Open Food Facts** | Supplementary barcode source | Free | No |
| **Jina Reader** (`r.jina.ai`) | Renders JS SPAs, returns clean text | Free | No |
| **Tavily** | Finds the official product page from a barcode lookup; gathers AUD price signals. **Only needed when using barcodes.** | Free 1,000/month | Optional — `tvly-...` key from [app.tavily.com](https://app.tavily.com) |
| **Ollama** | Local AI model host | Free | No |
| **Claude API** | Cloud AI (faster, supports image input) | Pay-per-use | Optional — key from [console.anthropic.com](https://console.anthropic.com) |
| **Shopify Admin API** | Creates draft products in your store | Free (requires store) | Admin API token |
| **eBay Trading API** | Creates scheduled (draft) listings in Seller Hub | Free (requires eBay Developer account) | App ID + Dev ID + Cert ID + User Token |
| **eBay Taxonomy API** | Auto-suggests category IDs for eBay listings | Free | Same eBay app credentials |

The app works without Tavily, without Shopify credentials, and without eBay credentials — you can paste a URL or HTML manually and copy the output instead of pushing.

## Environment variables

Copy `.env.example` to `.env` and fill in what you need. All variables are optional unless noted.

| Variable | Default | Description |
|---|---|---|
| `TAVILY_API_KEY` | _(none)_ | Enables auto product page search and AUD price signals. Can also be entered in Settings. |
| `OLLAMA_HOST` | `http://localhost:11434` | Ollama API base URL. Change if Ollama runs on a different port or machine. |
| `OLLAMA_MODEL` | `qwen3:8b` | Ollama model to use. |
| `CLAUDE_MODEL` | `claude-sonnet-4-6` | Claude API model used when a key is entered in Settings. |
| `EBAY_APP_ID` | _(none)_ | eBay app Client ID — required for eBay push and category suggestions. |
| `EBAY_DEV_ID` | _(none)_ | eBay Developer ID — required for eBay Trading API calls. |
| `EBAY_CERT_ID` | _(none)_ | eBay app Client Secret — required for eBay push and category suggestions. |

The **eBay User Token** and **Shopify** credentials are entered in the app's Settings panel and stored in browser localStorage only — never in `.env`.

## How it works

```
Image / Barcode / URL / HTML
          │
          ▼
1. Barcode lookup (if barcode provided)
          │  UPC Item DB + Open Food Facts (parallel)
          │  → product name, brand, category, price range
          │
          ▼
2. Product page search (if no URL yet)
          │  Tavily → finds the official manufacturer page
          │
          ▼
3. Page scrape
          │  Jina Reader (handles JS SPAs) → fallback: Cheerio + JSON-LD + meta
          │  Skipped entirely for image-only submissions
          │
          ▼
4. Price signals
          │  Tavily web search → Australian RRP mentions
          │
          ▼
5. AI generation  (platform-specific prompt)
          │
          ├─ Shopify tab → HTML description, AUD price, alt text
          │
          └─ eBay tab   → eBay title (≤80 chars), HTML description,
                          condition, item specifics, AUD price, alt text
          │
          ▼
6. Push (optional)
          ├─ Shopify Admin API → draft product (review in Shopify admin)
          └─ eBay Trading API  → scheduled listing (activate in Seller Hub)
```

## Project structure

```
src/
  types/index.ts              — shared TypeScript interfaces
  lib/
    barcode.ts                — UPC Item DB + Open Food Facts parallel lookup
    search.ts                 — Tavily search for official product page
    scraper.ts                — Jina Reader + cheerio fallback page scraper
    claude.ts                 — Ollama + Claude API streaming description generator (Shopify + eBay prompts)
    price.ts                  — multi-source AUD price signal gathering
    shopify.ts                — Shopify Admin API draft product creation
    ebay.ts                   — eBay Trading API listing creation + Taxonomy API category suggestions
  app/
    page.tsx                  — main UI — Shopify / eBay tab switcher (client component)
    api/
      lookup/route.ts         — POST endpoint orchestrating the full pipeline
      ollama-status/route.ts  — Ollama health check (polled every 15s by UI)
      tavily-usage/route.ts   — Tavily usage proxy
      shopify/route.ts        — Shopify draft product push endpoint
      ebay/route.ts           — eBay scheduled listing push endpoint
      ebay-categories/route.ts — eBay category suggestion endpoint
```
